# RF Analyzer — project context

## What this is
Simulated RF signal intelligence dashboard. Synthetic signals flow through
an ML classifier, get threat-scored, and stream via WebSocket to a React dashboard.
No real hardware. Fully self-contained.

## Signal schema (do not change once frontend is started)
{
  id: string,           // format: "sig_00001" (zero-padded counter)
  timestamp: ISO string,
  frequency_mhz: float,
  power_dbm: float,
  bandwidth_khz: float,
  modulation: "AM" | "FM" | "PSK" | "QAM",
  duration_ms: int,
  label: "friendly" | "unknown" | "hostile",
  confidence: float 0-1,
  threat_score: int 0-100
}

## Internal ML-only fields (stripped before WebSocket send)
These are generated by simulator.py and used as classifier features only:
  freq_hop_flag: int (0 or 1)
  burst_regularity: float 0-1

## Signal classes
- friendly:  136–174 MHz, AM/FM, power -80 to -60 dBm, regular bursts
- unknown:   300–512 MHz, mixed modulation, power -70 to -50 dBm, irregular
- hostile:   2000–4000 MHz, PSK/QAM, power -55 to -30 dBm, freq-hopping

## Threat score formula
power_weight = 1.0 if power_dbm > -50 else 0.7
score = int(confidence * power_weight * 100) if label == "hostile" else int(confidence * 20)
Alert threshold: score > 70

## Scenarios
- peacekeeping:    90% friendly, 8% unknown, 2% hostile
- border_patrol:   60% friendly, 25% unknown, 15% hostile
- active_conflict: 45% friendly, 30% unknown, 25% hostile

## Ports
- Production (Docker):  http://localhost (nginx on port 80)
- Backend direct:       http://localhost:8000
- Frontend dev server:  http://localhost:5173
- WebSocket (prod):     ws://localhost/ws/signals      (via nginx proxy)
- WebSocket (dev):      ws://localhost:5173/ws/signals  (via Vite proxy)

## WebSocket cadence: one event per 500ms (~2 signals/sec)

## Quick start
```
docker compose up
```
Model is pre-trained at image build time (RUN python train.py in backend/Dockerfile).

## REST API
GET  /scenario          → {"scenario": str, "options": [...]}
POST /scenario          → body {"name": str}, switches active scenario live
GET  /status            → {"status": "ok", "scenario": str, "clients_connected": int}
WS   /ws/signals        → streams signal events at 500ms cadence

## Stack
Backend:  FastAPI, scikit-learn, NumPy, Pydantic v2, uvicorn, joblib
Frontend: React 18, Vite, Tailwind CSS v3, Recharts
Proxy:    nginx (prod), Vite dev proxy (dev)
Infra:    Docker Compose, network rf-net

## Classifier features (10 total)
frequency_mhz, power_dbm, bandwidth_khz, duration_ms,
freq_hop_flag, burst_regularity,
mod_AM, mod_FM, mod_PSK, mod_QAM  (one-hot)

## Model
RandomForestClassifier(n_estimators=100, random_state=42)
Trained on 6000 signals (2000 per class), test accuracy 1.0000 on all three classes.
Saved to backend/model.pkl via joblib.

## Rules
- No real RF libraries (no GNU Radio, no SDR)
- No auth, no database — fully stateless
- Never change signal schema once frontend exists
- Keep all components in separate files
- freq_hop_flag and burst_regularity are ML features — never send them over WebSocket

## Frontend component list

### Hooks
- src/hooks/useSignalFeed.js   — WebSocket client; exposes signals[], threatScore, connected, totalSeen, signalsPerSec
- src/hooks/useCountUp.js      — rAF easeOutQuart counter animation; useCountUp(target, duration) → integer

### Context
- src/context/ThemeContext.jsx  — ThemeProvider + useThreatMode(); alertActive, alertSignal, triggerAlert(), clearAlert()

### Components
- src/components/AlertOverlay.jsx      — Sticky z-50 threat banner with glitch text + countdown; click to dismiss
- src/components/ScenarioSwitcher.jsx  — Three scenario buttons; calls POST /api/scenario
- src/components/MetricCard            — Inline in App.jsx; glass-panel card with animated value
- src/components/SpectrogramCanvas.jsx — HTML5 canvas waterfall; Gaussian blobs; ResizeObserver full-width
- src/components/FeedTable.jsx         — Scrollable signal table, last 50 rows, neon label badges
- src/components/ThreatChart.jsx       — Recharts AreaChart, last 30 signals, #ff2020 neon line
- src/components/RadarWidget.jsx       — Pure SVG radar; rotating sweep arm; friendly/hostile/unknown dots
- src/components/TerminalLog.jsx       — Typewriter terminal log; master 16ms ticker; 220px fixed height
